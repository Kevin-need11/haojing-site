開始前，確認你有這三個帳號

・Claude 帳戶：需付費方案（Pro 以上），並能安裝 Claude Code。
・Google 帳戶：用來跑 Apps Script 與放行事曆，平常的 Gmail 就可以。
・LINE 官方帳號：在 LINE Developers 建一個 Messaging API channel（下一頁教）。

照順序做，完全沒程式底子也能跟。前幾步在準備工具，第六步把整套交給 Claude

1

先打開「終端機」

後面很多步要在「終端機」（一個打指令的黑色小視窗）裡貼指令。先學會打開它：

  1. Windows：按左下角開始鍵、打「cmd」、按 Enter。
  2. Mac：按 ⌘ + 空白鍵、打「Terminal」、按 Enter。

之後所有「在終端機輸入」的指令，都是貼進這個視窗、按 Enter。

2

安裝 Node.js

Claude Code 要靠它才能安裝。到 nodejs.org 下載 LTS 版、一路下一步裝完。裝完後把終端機視窗關掉、重新打開一次（不然它抓不到），再輸入下面這行，有出現版本號（例如 v20.x）才繼續：

確認安裝
node -v
3

安裝 Claude Code

在終端機輸入：

安裝指令
npm install -g @anthropic-ai/claude-code
若出現 permission／EACCES 之類的紅字（權限不足）：Mac 使用者在指令最前面加 sudo 再跑一次（會要你輸入電腦開機密碼）。
4

打開 Apps Script API ＋ 授權

先到下面網址，把「Apps Script API」切成開啟（不開，之後 Claude 建專案會失敗）：

Apps Script 設定
https://script.google.com/home/usersettings

再讓工具有權部署（會開瀏覽器，用你的 Google 帳號按一次「允許」）：

授權指令
npx @google/clasp login
第一次跑會問你「是否安裝 clasp 套件？」，輸入 y 按 Enter 即可。
5

建立 LINE 官方帳號 ＋ Messaging API

  1. 到 developers.line.biz 用 LINE 登入 → 建一個 Provider。
  2. 建一個 Messaging API channel（會對應一個 LINE 官方帳號）。
  3. Messaging API 分頁 → 捲到最下面「Channel access token」區、按 Issue 發行，複製這串 長期 token。
  4. Basic settings 分頁：記下 Channel secret。
  5. 到 LINE Official Account Manager → 回應設定：關掉「自動回應訊息」與「歡迎訊息」、打開「Webhook」。
不關自動回應，客戶只會收到 LINE 內建罐頭回覆，會誤以為壞掉。
6

讓 Claude 串接上（重點）

這一步你要做的很單純，其餘交給 Claude：

  1. 在終端機輸入 claude 開一個新的 Claude Code 對話，把「提示詞」那頁的主指令整段貼進去。
  2. Claude 會反問你要四個字串，直接在這個 Claude 對話視窗裡把它們貼回去：
    GEMINI_API_KEY CALENDAR_ID LINE_CHANNEL_ACCESS_TOKEN LINE_CHANNEL_SECRET
  3. Claude 會自動建程式、部署成網頁應用程式、把網址接上 LINE webhook。
  4. 最後一步一定要你自己做：照 Claude 的指示回到 Apps Script 編輯器、執行一次 setup 函式，這時會跳出授權畫面，按「允許」（授權讀寫行事曆與對外連線）。
★ 第 4 小點最容易被漏掉：不回編輯器跑一次 setup 並按「允許」，截圖傳進來時程式會因為沒授權而沒反應。看似部署好卻不動，通常就是卡在這。
7

測試與驗證

先加自己的官方帳號好友：在 LINE Developers 的 Messaging API 分頁有一個 QR code，用手機 LINE 掃它，就能把你剛建的官方帳號加為好友。然後傳一張「對話截圖」給它。收到「事件／時間／地點／已排行事曆」的整理卡，就代表串接成功。

若 Claude 跑 LINE 的 webhook 驗證時顯示失敗，別緊張——那常是雲端第一次啟動較慢造成，以「實際傳一張截圖有沒有回卡」為準。

Claude Code 設定完，把這一整段「主指令」貼給它，它會自動建好整套

這段已內建關鍵修正：先把今天日期與台北時區餵給 AI（否則「明天／下週三」會算錯）、一張圖支援多筆待辦、正確處理 LINE 取圖與回覆、讀錯可回一句話糾正，並在部署前完成授權。金鑰只存進系統屬性、不寫死在程式碼裡。

主指令（貼給 Claude Code）
# 任務：幫我自動建置並部署一套「LINE AI 行政助手」

你是一位資深自動化工程師。請幾乎全自動地幫我建好一套系統：
「LINE 傳截圖 → Gemini 讀懂 → 抽出待辦(誰/做什麼/何時) → 寫進 Google 行事曆 → 用 LINE 回覆確認卡」，
並部署上線、接好 LINE webhook。全程用繁體中文白話跟我講，每一步先說你要做什麼。

## 鐵律
1. 需要我手動的地方，要明確、逐步、附網址地請我做，等我回「好了」再繼續，絕不假裝已完成。
2. 金鑰只寫進 Apps Script 的「指令碼屬性」，永遠不寫死在程式碼、不印在對話裡。
3. 每完成一個里程碑就回報一句進度。

## 第 0 步：前置確認（請我先做完再繼續）
- 確認我已安裝 Node.js（請我跑 `node -v` 有版本號）。
- 請我到 https://script.google.com/home/usersettings 打開「Apps Script API」。
- 請我跑 `npx @google/clasp login`，瀏覽器按「允許」。等我回報成功再往下。

## 第 1 步：收集金鑰（一次問齊，做基本格式檢查）
GEMINI_API_KEY、CALENDAR_ID、LINE_CHANNEL_ACCESS_TOKEN、LINE_CHANNEL_SECRET。
CALENDAR_ID：用主行事曆就填我的 Gmail；若是自建行事曆，請我去行事曆設定複製那串
以 @group.calendar.google.com 結尾的 ID（否則 getCalendarById 會回 null）。

## 第 2 步：建立專案骨架（在 ~/line-admin-assistant）
- 寫 appsscript.json：時區 "Asia/Taipei"；webapp.access "ANYONE_ANONYMOUS"、
  executeAs "USER_DEPLOYING"；oauthScopes 含 script.external_request 與 calendar。
  ★因為 executeAs 是「我本人」，且 webhook 由 LINE 匿名打進來、匿名呼叫者無法代為授權，
  所以行事曆與對外連線權限，必須由我本人在部署前先手動授權一次（見第 4 步）。
- 執行 `npx @google/clasp create --type standalone --title "我的LINE行政助手"`。

## 第 3 步：寫主程式 Code.gs
- doPost(e)：解析 LINE events[]；對 message.type==="image" 的事件：
  用 messageId 呼叫 https://api-data.line.me/v2/bot/message/{id}/content（帶 Bearer token）取圖 →
  Utilities.base64Encode。讀圖 → 有日期就寫行事曆 → 回覆確認卡。最後一律回 HTTP 200。
- 回覆策略（★replyToken 約 1 分鐘會失效，Gemini 讀圖可能較慢）：
  先用 replyToken 秒回一則「收到，正在幫你整理…」；整理完的確認卡改用 push message
  （POST https://api.line.me/v2/bot/message/push，to 帶 event.source.userId）補送，避免逾時收不到。
- readScreenshot(base64)：呼叫 gemini-2.5-flash generateContent，temperature 0.2，
  responseMimeType application/json。送圖「之前」先用
  Utilities.formatDate(new Date(),"Asia/Taipei",...) 把今天日期與星期塞進提示詞
  （★沒這步相對日期會全錯）。用下方 PROMPT_READ，回「陣列」(可 0/1/多筆)，沒待辦回 []。
- 防護：candidates 可能為空或非法 JSON，包 try/catch，壞了回 []，不要整支當掉。
- createCalendarEvent(task)：CalendarApp.getCalendarById(CALENDAR_ID).createEvent(...)，
  addPopupReminder(120)。
- 寫一個 setup() 把四個金鑰寫進 PropertiesService.getScriptProperties()。

## PROMPT_READ（原樣使用，勿塞任何真實客戶資料）
你是嚴謹的行政助理，只負責抄寫。今天是 {{TODAY}}（{{WEEKDAY}}），時區 Asia/Taipei。
看這張截圖，抽出所有待辦，回傳 JSON 陣列，每筆：
{ "title":"誰＋要做什麼", "date":"YYYY-MM-DD，用今天換算明天/下週三等",
  "time":"HH:MM 24小時制，沒寫留空", "location":"沒有留空", "note":"帶什麼/提前多久",
  "confidence":"high 或 low" }
規則：沒有待辦回 []；看不清楚的欄位留空不要猜；人名地址照原文；只回 JSON 陣列。

## 第 4 步：注入金鑰 + 完成授權（這步一定要我本人動手，請明確帶我做）
- clasp push 上傳程式。
- 請我到 https://script.google.com 打開這個專案 → 選 setup 函式 → 按「執行」→
  這時會跳出授權畫面，請我按「允許」（一次同時完成金鑰注入與行事曆/外連授權）。
  ★不要叫我用 clasp run 注入（那需要另接 GCP 專案，我做不到）；一律用「編輯器手動跑 setup」。
- 授權完成後才 `npx @google/clasp deploy` 取得 /exec 網址；日後更新都用同一個
  deploymentId（clasp deploy -i ）以保持網址不變。

## 第 5 步：接 LINE webhook + 交付
- 呼叫 PUT https://api.line.me/v2/bot/channel/webhook/endpoint（帶 Bearer token、body 放 /exec 網址）
  自動接上 webhook，再 POST .../webhook/test 驗證；若 test 回非 200 也先別判失敗，以實際傳圖為準。
- 提醒我在 LINE console 開「Use webhook」、關「自動回應」。
- 印出 Web App 網址、LINE 加好友方式、和「傳一張截圖測試」的指引；提醒我把金鑰從對話清掉。

串好之後，整條龍是這樣運作的

1

你傳截圖到 LINE

把對話、預約訊息、報價單截圖，傳給你的 LINE 官方帳號。

2

訊息送進 Apps Script

LINE 把訊息事件送到你部署的 Google Apps Script（免費雲端程式）。

3

Gemini 讀圖、抽出待辦

AI 看懂截圖，抓出「誰、做什麼、何時」，並以今天日期把「明天、下週三」換算成實際年月日。

4

寫進行事曆並設定提醒

自動建立 Google 行事曆事件，並設定提前兩小時提醒，不必自己記。

5

LINE 回你一張確認卡

回覆整理好的卡片給你確認；若 AI 讀錯，你只要回一句「第二件改成週五下午三點」，它就會修正。
